My reading streak shows 0 even though I've been finishing books every day this week. The books show up in my history fine — just the streak is wrong

stats.py - route
stats_service.py  - this contains the streak logic

What fields does ReadingEvent have? Which ones are always set, and which can be null?
id, user_id, book_id, started_at, finished_at
Except finished_at, all are set. finished_at can be nullable initially and later can be filled out if needed.

What does the UniqueConstraint on ReadingEvent prevent?
It prevents the database from storing more than one reading event for the same user and book combination.

User has a reading_streak column. Does the stats endpoint use it?
No, it doesn't use that column. 


In routes.py - 
reading_streak    (int) — consecutive days with at least one book finished
books_this_month  (int) — books finished in the current calendar month
total_pages_read  (int) — total pages across all finished books


calculate_streak(user_id: str) -> int - takes the user_id, returns the streak_count
books_this_month(user_id: str) -> int - takes the user_id and returns the number of books read this month
total_pages_read(user_id: str) -> int - takes the user_id and returns the sum the page counts of all books the user has finished.

get_reading_history(user_id: str) -> list[ReadingEvent]:
docstring promise about the finished_at in descending order. But the actual code is ordering by started_at in descending order
